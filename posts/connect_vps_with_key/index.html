通过密钥来连接 vps，可以增强按照安全性，避免许多爆破尝试，比如以下的

生成密钥对#

NAME_IDRSA="xxx@xxx.nas"
FILE_IDRSA="$HOME/.ssh/xxx_rsa"

ssh-keygen -t rsa -C ${NAME_IDRSA} -f ${FILE_IDRSA}

编辑config#

# vim /root/.ssh/config

Host github.com
    HostName github.com
    PreferredAuthentications publickey
    IdentityFile /root/.ssh/github_rsa

Host gitea
    HostName xxx.xx
    Port yyyy
    User git
    IdentityFile C:/Users/xx/.ssh/xxx.xx_rsa

上传到 VPS#

中间需要输入密码，然后就会在~生成.ssh/authorized_keys, 并自动帮助设置好文件夹权限和 pubkey 的权限1

编辑 sshd 配置并且重启 sshd#

vim /etc/ssh/sshd_config


# 编辑以下行，取消掉前面的`#`
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
# 禁用密码登录
PasswordAuthentication no


systemctl sshd restart

使用 key 来连接，如果成功了，说明安装操作均没有问题。 否则，需要再次进行检查，哪一步出了问题。

系统重装#

  1. 重装前，编辑sshd_config, PasswordAuthentication修改为yes
  2. 重装系统，然后使用root密码登录
  3. 本地客户端，ssh-keygen -R hostname, hostname即ip地址，解除配对2
  4. 重新上传密钥，然后测试登录
